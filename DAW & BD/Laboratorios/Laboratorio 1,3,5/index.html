HTML:

  1. ¿Cuál es la diferencia entre Internet y la World Wide Web?
    • Segun se menciona en un articulo de BBC News(2019): El internet es un conjunto de computadoras conectadas entre si, alrededor del todo el mundo. Por otro lado, la WEB es una enorme coleccion de paginas que se encuentran dentro de las computadoras.
  2. ¿Cuáles son las partes de un URL?
    • "Protocolo HTTPS - Subdominio - Dominio - TLD - Ruta - Parámetro - Etiqueta" (Pukocz, s.f.).
  3. ¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?
    • GET:

      Segun Reichert (2017): El método GET solicita la transferencia de una representación seleccionada actual para el recurso de destino. GET es el mecanismo principal de recuperación de información.

    • POST:

      Segun Reichert (2017): El método POST solicita que el recurso de destino procese la representación incluida en la solicitud de acuerdo con la semántica específica del recurso.

    • PUT:

      Segun Reichert (2017): El método PUT solicita que el estado del recurso de destino sea creado o reemplazado por el estado definido por la representación incluida en la carga útil del mensaje de solicitud.

    • DELETE:

      Segun Reichert (2017): El método DELETE solicita que el servidor de origen elimine la asociación entre el recurso de destino y su funcionalidad actual.

    • PATCH:

      Segun Reichert (2017): La diferencia entre las solicitudes PUT y PATCH se refleja en la forma en que el servidor procesa la entidad adjunta para modificar el recurso identificado por la Request-URI. En una solicitud PUT, la entidad adjunta se considera una versión modificada del recurso almacenado en el servidor de origen y el cliente solicita que se reemplace la versión almacenada. Sin embargo, con PATCH, la entidad adjunta contiene un conjunto de instrucciones que describen cómo se debe modificar un recurso que reside actualmente en el servidor de origen para producir una nueva versión.

  4. ¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?
    • POST
  5. ¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?
    • GET
  6. Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

    • "Este código de estado indica que la acción solicitada por el cliente ha sido recibida, entendida, aceptada y procesada correctamente. 200 OK" (Lázaro, 2018).

  7. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

    • No necesariamente, depende si el usuario estaba accediendo a algo que tenga permitido o simplemente estaba intentando acceder a algun lugar que de verdad no tiene existencia. Es decir, es culpa del usuario.

  8. ¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

    • Si, ya que es un error en el servidor, y se necesita encontrar el problema desde adentro y ser resuelto.

  9. ¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

    • Significa que ya son viejos, y no es recomendado usarlos. Como por ejemplo: basefont, center, font, strike, etc.

  10. ¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

    • Sintaxis

      "El término HTML posee una sintaxis compatible con HTML4 y XHTML1 publicados en la red, pero no compatible con las características más esotéricas del SGML de HTML4" (Canjura, s.f.).

    • Modelos Estrictos de Contenido

      "HTML5 define de forma más estricta el contenido para elementos como "div". Estos elementos ahora pueden contener contenido de elementos «block» o «inline» pero no los dos. En HTML4 esto fue considerado como un bug de la espeficación ya que permitía el uso de ambos. En cambio HTML5, hace la utilizacion de JavaScript y CSS para llevar una organizacion mas pulcra" (Canjura, s.f.).

    • Caption - Thread - Tbody - Tfoot
  12. ¿Cuáles son los principales controles de una forma HTML5?
    • Control - Form - Legend - Textarea - Label - Button - Fieldset

    Sobre el ciclo de vida y desarrollo de los sistemas de información:
  14. ¿Cuál es el ciclo de vida de los sistemas de información?

    • "El ciclo de vida de un sistema de información es un enfoque por fases del análisis y diseño que sostiene que los sistemas son desarrollados de mejor manera mediante el uso de un ciclo especifico de actividades del analista y del usuario" (Cadena,2014).

  15. ¿Cuál es el ciclo de desarrollo de sistemas de información?

    • "El ciclo de vida de desarrollo de un sistema (SDLC) es un proceso lógico utilizado en el mundo del Desarrollo de Software sistemas para desarrollar un sistema de información, incluidos los requisitos, la validación, formación, como los usuarios (interesados) en la propiedad" (University of Science and Technology, 2019).

CSS:

  1. Como ingeniero de software ¿cuál es tu recomendación sobre el uso de !important en un CSS?

    • Cuando se presente un caso muy importante, en el que necesites un estilo en cierto lugar, que ya tienes configurado. Pero si es posible, no usarlo es mejor, ya que en CSS ya tiene jerarquias que puedes usar, para evitar usar !important.

  2. Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

    • Porque si la imagen esta en otro lado, es posible que algun dia amanesca sin fondo, por lo que es muy importante tener una imagen alt configurada, por si esto pasa, tu probablemente seguira teniendo fondo.

  3. Como ingeniero de software, ¿cuál es tu recomendación al elegir las unidades de un propiedad de estilo entre %, px y pt?
    • Depende de los usuarios, y el numero de pixeles en su pantalla, por lo que es mucho mejor utilizar % para que sea más relativo.
  4. ¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

    • Le ahorra tiempo al navegador del usuario en descargar el Stylesheet, por lo que hace de la página más rapida.
